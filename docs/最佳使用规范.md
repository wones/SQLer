# 个人SQL助手 - 最佳使用规范

## 一、命名规范

### 1.1 别名命名规则

#### 基础格式
```
{业务模块}_{查询类型}[_{修饰符}]
```

#### 示例
| 用途 | 命名示例 | 说明 |
|------|---------|------|
| 用户基础表 | `user_base` | 业务_类型 |
| 用户订单查询 | `user_order_list` | 业务_类型_列表 |
| 活跃用户查询 | `user_active_daily` | 用户_活跃_日 |
| 用户订单统计 | `user_order_count_monthly` | 用户_订单_统计_月 |

#### 命名约定
| 元素 | 规则 | 示例 |
|------|------|------|
| 分隔符 | 使用下划线 `_` | `user_order` ✅ |
| 禁用字符 | 避免特殊字符、空格 | `user-order` ❌ |
| 大小写 | 推荐小写 | `user_order` ✅ |
| 长度 | 建议 3-30 字符 | `u_order_2024` |
| 语义化 | 使用有意义的词汇 | `order_pending` ✅ |

### 1.2 分组命名规则

```
{业务域}_{层级}
```

| 示例 | 说明 |
|------|------|
| `crm_customers` | CRM客户域 |
| `crm_customers_domestic` | CRM-国内客户 |
| `crm_customers_overseas` | CRM-海外客户 |
| `bi_reports` | BI报表域 |
| `bi_reports_daily` | BI-日报 |
| `bi_reports_monthly` | BI-月报 |

### 1.3 表命名规则

```
{业务缩写}_{表用途}
```

| 示例 | 说明 |
|------|------|
| `sys_user` | 系统-用户表 |
| `sys_role` | 系统-角色表 |
| `sys_permission` | 系统-权限表 |
| `ord_order` | 订单-订单表 |
| `fin_payment` | 财务-支付表 |

### 1.4 列命名规则

```
{表缩写}_{列含义}
```

| 示例 | 说明 |
|------|------|
| `user_id` | 用户ID |
| `user_name` | 用户名 |
| `user_create_time` | 创建时间 |
| `order_amount` | 订单金额 |
| `order_status` | 订单状态 |

## 二、别名编写规范

### 2.1 SQL模板规范

#### ✅ 规范的SQL模板
```sql
-- 别名: user_order_summary
-- 描述: 用户订单汇总查询
SELECT 
    u.user_id,
    u.user_name,
    u.user_phone,
    COUNT(o.order_id) AS order_count,
    SUM(o.order_amount) AS total_amount
FROM sys_user u
LEFT JOIN ord_order o ON u.user_id = o.user_id
WHERE o.order_status = 'PAID'
GROUP BY u.user_id, u.user_name, u.user_phone
```

#### ❌ 不规范的SQL模板
```sql
-- 问题1: 没有注释
SELECT id, name FROM user

-- 问题2: 缺少表前缀
SELECT id, name FROM user WHERE id = 1

-- 问题3: SELECT * (生产环境禁用)
SELECT * FROM user

-- 问题4: 缺少WHERE条件
SELECT id, name FROM user
```

### 2.2 列定义规范

每个别名必须正确定义列信息，包括：

| 字段 | 说明 | 示例 |
|------|------|------|
| 列名 | 字段名称 | `user_name` |
| 类型 | 数据类型 | `VARCHAR` |
| 注释 | 业务含义 | `用户名` |

**示例**：
```
┌────────────┬─────────┬────────────┐
│ 列名       │ 类型    │ 注释       │
├────────────┼─────────┼────────────┤
│ user_id    │ BIGINT  │ 用户ID     │
│ user_name  │ VARCHAR │ 用户姓名   │
│ order_cnt  │ INT     │ 订单数量   │
│ total_amt  │ DECIMAL │ 消费总额   │
└────────────┴─────────┴────────────┘
```

### 2.3 依赖表定义规范

必须准确填写依赖的表和别名：

```json
{
  "table_dependencies": [
    "sys_user",
    "ord_order"
  ]
}
```

## 三、高级用法

### 3.1 别名链式组合

#### 层级组合示例

```
┌─────────────────────────────────────────────────────────────┐
│  Level 1: 基础原子查询                                        │
├─────────────────────────────────────────────────────────────┤
│  user_base      → SELECT id, name FROM sys_user            │
│  order_base     → SELECT user_id, amount FROM ord_order    │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Level 2: 组合查询                                           │
├─────────────────────────────────────────────────────────────┤
│  user_order     → SELECT u.name, o.amount                  │
│                   FROM user_base u                          │
│                   JOIN order_base o ON u.id = o.user_id     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Level 3: 业务查询                                           │
├─────────────────────────────────────────────────────────────┤
│  user_order_summary → SELECT * FROM user_order              │
│                        WHERE amount > 1000                 │
└─────────────────────────────────────────────────────────────┘
```

#### 组合原则

| 原则 | 说明 |
|------|------|
| 原子化 | 基础查询尽量简单、单一功能 |
| 可复用 | 原子查询可被多个组合查询引用 |
| 语义化 | 每个层级都有明确的业务含义 |
| 可追溯 | 依赖关系清晰可见 |

### 3.2 分层架构实践

```
┌─────────────────────────────────────────────────────────────┐
│                    报表/展示层                              │
│  user_monthly_report  │  order_daily_report                 │
```

## 五、最佳实践案例

### 5.1 用户分析场景

```sql
-- 基础层: 用户基础信息
-- 别名: user_base
-- 依赖: sys_user
SELECT 
    user_id,
    user_name,
    user_phone,
    user_email,
    register_time
FROM sys_user
WHERE status = 1

-- 聚合层: 用户订单汇总
-- 别名: user_order_agg
-- 依赖: user_base, ord_order
SELECT 
    u.user_id,
    u.user_name,
    COUNT(o.order_id) AS order_count,
    SUM(o.order_amount) AS total_amount,
    MAX(o.create_time) AS last_order_time
FROM user_base u
LEFT JOIN ord_order o ON u.user_id = o.user_id
GROUP BY u.user_id, u.user_name

-- 业务层: 用户价值分析
-- 别名: user_value_analysis
-- 依赖: user_order_agg
SELECT 
    user_id,
    user_name,
    order_count,
    total_amount,
    CASE 
        WHEN total_amount > 10000 THEN '高价值'
        WHEN total_amount > 5000 THEN '中价值'
        ELSE '普通用户'
    END AS user_level
FROM user_order_agg
```

### 5.2 订单分析场景

```sql
-- 基础层: 订单基础查询
-- 别名: order_base
-- 依赖: ord_order, sys_user
SELECT 
    o.order_id,
    o.order_no,
    o.user_id,
    u.user_name,
    o.order_amount,
    o.order_status,
    o.create_time
FROM ord_order o
JOIN sys_user u ON o.user_id = u.user_id
WHERE o.is_deleted = 0

-- 聚合层: 订单状态统计
-- 别名: order_status_stats
-- 依赖: order_base
SELECT 
    order_status,
    COUNT(*) AS order_count,
    SUM(order_amount) AS total_amount
FROM order_base
GROUP BY order_status

-- 业务层: 订单日报
-- 别名: order_daily_summary
-- 依赖: order_base
SELECT 
    DATE(create_time) AS stat_date,
    COUNT(*) AS order_count,
    SUM(order_amount) AS total_amount,
    AVG(order_amount) AS avg_amount
FROM order_base
GROUP BY DATE(create_time)
ORDER BY stat_date DESC
```

## 六、常见问题处理

### 6.1 列名冲突

当多个表的列名相同时，使用表别名区分：

```sql
-- 错误示例
SELECT id, name FROM user, order
-- id 和 name 产生歧义

-- 正确示例
SELECT u.id, u.name, o.id, o.name 
FROM sys_user u, ord_order o
```

### 6.2 性能优化

| 问题 | 解决方案 |
|------|---------|
| 全表扫描 | 添加索引条件 |
| SELECT * | 只查询需要的列 |
| 子查询过深 | 使用 CTE 或临时表 |
| JOIN 过多 | 精简关联、必要时拆分 |

### 6.3 依赖循环

避免循环依赖：

```
❌ 循环依赖
user_order → user_base → order_detail → user_order

✅ 单向依赖
user_base → user_order → user_order_summary
```
